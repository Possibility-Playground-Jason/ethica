Ethica

AI ethics compliance checking as a service, powered by the UNESCO Recommendation on the Ethics of AI.

Check a Repository

Paste any public GitHub repo URL to run an ethics compliance check.

    API Endpoints

    MethodPathDescription
    GET/healthHealth check
    GET/frameworksList available frameworks
    POST/checkRun compliance check (JSON results)
    POST/check/reportRun compliance check (HTML report)
    POST/check/badgeRun compliance check (SVG badge)
    GET/badge/:owner/:repoREADME-embeddable SVG badge

    POST /check

    Run a full compliance check against a public GitHub repository and return structured JSON results.

    Request Body

    FieldTypeDescription
    repo_urlstring required GitHub repository URL (e.g. https://github.com/owner/repo.git)
    refstring optional Branch or tag to check (defaults to the repo's default branch)
    frameworkstring optional Framework ID (default: unesco-2021)
    compliance_levelstring optional One of basic, standard, or verified (default: standard)

    Example

    curl -X POST https://ethica.jason-266.workers.dev/check \
  -H "Content-Type: application/json" \
  -d '{
    "repo_url": "https://github.com/owner/repo.git"
  }'

    With optional parameters:

    curl -X POST https://ethica.jason-266.workers.dev/check \
  -H "Content-Type: application/json" \
  -d '{
    "repo_url": "https://github.com/owner/repo.git",
    "ref": "main",
    "framework": "unesco-2021",
    "compliance_level": "verified"
  }'

    POST /check/report

    Same request body as /check, but returns a rendered HTML report card instead of JSON. Useful for embedding in iframes or opening directly in a browser.

    curl -X POST https://ethica.jason-266.workers.dev/check/report \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/owner/repo.git"}' \
  -o report.html

    POST /check/badge

    Same request body as /check, but returns an SVG badge image. Useful for generating a badge dynamically via a POST request.

    curl -X POST https://ethica.jason-266.workers.dev/check/badge \
  -H "Content-Type: application/json" \
  -d '{"repo_url": "https://github.com/owner/repo.git"}' \
  -o badge.svg

    GET /badge/:owner/:repo

    Returns an SVG compliance badge for a GitHub repository. This is the simplest way to add an Ethica badge to your README since it works as a plain image URL.

    Query Parameters

    ParamDefaultDescription
    levelstandardCompliance level: basic, standard, or verified
    frameworkunesco-2021Framework ID
    refdefault branchBranch or tag to check

    Compliance Levels

    LevelMin Pass RateScope
    basic50%Transparency + Accountability only
    standard70%+ Fairness, Privacy, Safety
    verified95%All principles

    What It Checks

    Ethica runs automated checks across 6 of the 10 UNESCO AI ethics principles:

    PrincipleCheckWhat it looks for
    TransparencyModel / System CardMODEL_CARD.md or SYSTEM_CARD.md
    TransparencyExplainabilityshap, lime, captum, etc. in deps
    TransparencyProvider TransparencyAI providers that publish system cards
    FairnessFairness Metricsfairlearn, aif360, etc. in deps
    PrivacyPrivacy AssessmentPRIVACY_IMPACT_ASSESSMENT.md or PRIVACY.md
    AccountabilityVersion Control.git/ directory
    AccountabilityREADMEREADME.md (or .rst, .txt)
    SafetyDependency Manifestpackage.json, requirements.txt, etc.
    SafetyLicenseLICENSE, COPYING, etc.
    SustainabilityEthical Impact DocETHICS.md or ETHICAL_IMPACT_ASSESSMENT.md
    Note: Ethica uses the GitHub API to inspect repositories — no code is cloned or executed. Public repos work out of the box. Rate limits apply (60 req/hr unauthenticated, 5,000/hr with a token).

    CLI Usage

    Ethica is also available as a Python CLI for local checks:

    pip install -e "."
cd your-project
ethica init       # creates .ai-ethics.yaml + doc templates
ethica check      # run compliance checks

    See the GitHub repository for full documentation.
